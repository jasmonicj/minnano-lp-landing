## 構築期間の定義

- **ヒアリングシート送付日 = 0日目**
- **0〜7日目（構築期間）**：人的サポートによる初期LP構築。スタッフが対応。
- **8日目〜（月額保守フェーズ）**：LINEによる自動修正対応を開始。月額課金もこのタイミングで開始。
- クライアントが7日以内に初期LPの承認（OK）をした場合は、その時点でmaintenanceに移行する（7日を待つ必要はない）。
- 7日経過してもOKが出ない場合は、report-cron（月曜実行）がスタッフLINEに「⏰ 構築期間7日超過：[clientName]」を通知する。

## 13. サービス範囲定義（20260313確定版）

> **【設計上の制約】LINE 1アカウント = 1担当者**
> 本サービスはLINE公式アカウントの仕様上、1クライアントにつき1つのLINEアカウントからのみ操作可能。
> グループチャットや複数メンバーからの同時操作には対応しない。
> 担当者変更時は店長に連絡し `scripts/change-line-user.ts` で対応する。

### 修正回数
| 項目 | 定義 |
|------|------|
| 月間修正依頼回数 | **上限なし（使い放題）** |
| 新規セクション追加 | **AI対応範囲内なら無制限・無償** |
| 人的サポート必要案件 | **月1件無償・2件目以降¥5,000/件** |

### セキュリティ要件（実装必須）

#### 1. メッセージングAPIのアクセススコープ制限
| 項目 | 要件 |
|------|------|
| Webhook署名検証 | LINE Messaging APIのX-Line-Signatureを全リクエストで必須検証 |
| 送信元制限 | LINEのIPアドレスレンジからのみWebhookを受け付ける |
| アクセストークン | Webhookハンドラーはメッセージ送受信スコープのみ付与（管理API等は別スコープ） |
| ユーザーID検証 | Vercel KVに登録済みのuserId以外からのメッセージは処理しない |

#### 2. プロンプト生成時のサニタイズ
| 項目 | 要件 |
|------|------|
| インジェクション対策 | ユーザーメッセージをプロンプトに組み込む前に「指示語」を無害化（例：「ignore previous」「あなたはxxxです」等を検出・ブロック） |
| ロールの固定 | system promptのroleは常にfixedとし、ユーザー入力でroleを上書きできない構造にする |
| 出力検証 | AIの出力がHTML/CSS/JS以外の形式（シェルコマンド等）を含む場合は実行前にブロック |
| 最大文字数 | ユーザーメッセージは1000文字上限・超過分はトリム後に処理 |

#### 3. コード実行・デプロイの安全性
| 項目 | 要件 |
|------|------|
| ソースコード非公開 | クライアントにはリポジトリアクセス権を付与しない（契約終了時にZIP渡し） |
| デプロイ権限 | GitHub ActionsのTokenはWrite権限を最小スコープ（対象リポジトリのcontentsのみ）に限定 |
| Secrets管理 | API Key類はVercel Environment Variables / GitHub Secretsに格納・コードにハードコードしない |
| 依存ライブラリ | Dependabotを有効化・高リスク脆弱性は48時間以内に対応 |

#### 4. クライアントデータ保護
| 項目 | 要件 |
|------|------|
| memory隔離 | 各クライアントの記憶ファイルは必ずclient-{id}単位で分離・他クライアントのデータを誤参照しない構造にする |
| ログ管理 | LINEメッセージ本文はサーバーログに残さない（userId・タイムスタンプのみ記録） |
| 通信暗号化 | 全通信はHTTPS必須・HTTP→HTTPSリダイレクト強制 |

#### 5. インジェクション検知・BAN処理
| 項目 | 要件 |
|------|------|
| 検知 | インジェクション疑いのあるメッセージを自動フラグ・店長LINEに即時通知 |
| 警告 | 1回目：自動で警告メッセージをクライアントに送信・フラグをmemoryに記録 |
| BAN判定 | 警告後に再検知 → 当社専門委員会の審議を経て、Vercel KVのステータスを`banned`に変更 |
| BAN後処理 | 以降のWebhookを全て無視・Stripeの定期課金を停止（残存期間分は一括請求） |
| 成果物 | BANクライアントのリポジトリはprivate維持・ZIP渡し不可 |
